## Phase 3 Gap Fill Plan (configure-driven)

Purpose: turn every failed probe from `plan.md` into actionable work items with the subsystem and file roots that should carry the fixes. Nothing below writes code yet—it is a backlog for kernel (`crates/*`, `kernel/src/*`), libc (`userspace/libc`), and the `external/cpython` port.

- Fix toolchain plumbing so configure can locate the linkers and platform hooks (`GNU ld input path`, `x86_64-unknown-linux-gnu-llvm-profdata`, `MACHDEP_OBJS`, `library containing opendir`, `library containing sem_init`, `library containing shm_open`, `library containing yp_match`, `library containing login_tty`). This lives in `toolchain/`, `build/Makefile`, and the cross binutils wrappers.
- Settle ABI/platform switches inside `external/cpython-oxide-config.site` and `userspace/libc/include` for `__fpu_control`, `__fpu_control in -lieee`, `digit size for Python's longs`, `oxide-cc -pthread options needed to detect all undeclared functions`, `x87-style double rounding`, `if --enable-ipv6 is specified`, `if PTHREAD_SCOPE_SYSTEM is supported`, `if sockaddr has sa_len member`, `whether _XOPEN_SOURCE should be defined`, `whether byte ordering is bigendian`, `whether float word ordering is bigendian`, `whether oxide-c++ also accepts flags for thread support`, `whether oxide-cc -fno-strict-aliasing accepts and needs -fno-strict-aliasing`, `whether oxide-cc accepts -Kpthread`, `whether oxide-cc accepts -Kthread`, `whether pthreads are available without options`, `whether to enable large file support`, `whether wchar_t is usable`, `whether we can use gcc inline assembler to get and set mc68881 fpcr`.
- Document deliberate platform assumptions such as `the Android API level` check (make configure short-circuit to "not Android") and provide a deterministic path for `GNU ld input path` inside our build scripts.

### Process lifecycle, scheduling, credentials, namespaces (`crates/proc/proc`, `kernel/src/scheduler.rs`, `userspace/libc/src/process.rs`)
